### 1.2 守恒物理量

**守恒变量（Conservative Variables）**：直接参与守恒方程的变量，由显存持久化存储。

以下的密度均为相对体积而言的密度。

| 代码变量名 | 数学符号 | 含义详解 (位置、方向、依赖关系)                              | 来源 (生成者) | 用途 (消费者) |
| ---------- | -------- | ------------------------------------------------------------ | ------------- | ------------- |
| `rho`      | $\rho$   | **【位置】中心** <br/> **【含义】** 方格内气体密度。 <br/> **【依赖】** 由显存存储，作为整个求解器维护的对象。 |               |               |
| `rho_u`    | $\rho u$ | **【位置】中心** <br/> **【含义】** x轴方向动量密度。 <br/> **【依赖】** 由显存存储，作为整个求解器维护的对象。 |               |               |
| `rho_v`    | $\rho v$ | **【位置】中心** <br/> **【含义】** y轴方向动量密度。 <br/> **【依赖】** 由显存存储，作为整个求解器维护的对象。 |               |               |
| `E`        | $E$      | **【位置】中心** <br/> **【含义】** 总能量密度（内能+动能）。 <br/> **【依赖】** 由显存存储，作为整个求解器维护的对象。 |               |               |

### 2.6 CUDA 并行计算基础概念

- **Thread (线程)**：最小执行单位，对应一个网格单元。
- **Warp**：32 个线程为一组，GPU 的最小调度单位。X 维度对齐 warp 宽度（32）可实现最优合并访存。
- **Block (线程块)**：一组线程，共享一块快速的共享内存（Shared Memory），并且可以同步。本项目使用 32×8 = 256 线程的线程块。
- **Grid (网格)**：所有 Block 的集合，覆盖整个计算任务（整个物理场）。
- `threadIdx.x/y`：线程在 Block 内的坐标。
- `blockIdx.x/y`：Block 在 Grid 内的坐标。
- `blockDim.x/y`：一个 Block 的尺寸（当前为 32×8）。

## 3. 2026/2/8 重构优化日志

本次重构主要修正了 Riemann 求解器的物理错误，并针对计算链进行了深度优化，消除了大量冗余计算与内存分配开销。

### 3.1 物理与逻辑修正

1.  **Riemann 求解器修正**：
    *   **HLLC X 方向**：修正 Roe 声速计算，补回遗漏的垂直速度分量 $v$。
    *   **HLL Y 方向**：修正 Roe 声速计算，补回遗漏的水平速度分量 $u$。
    *   **HLLC Y 方向**：完全重写。原实现未使用 Roe 平均，现统一为标准的 Roe 平均计算，保证 XY 方向精度一致。
2.  **马赫数统计修正**：
    *   旧逻辑错误使用全局来流声速作为分母：$Ma = (|v|+c)/c_{\infty}$。
    *   新逻辑修正为局部马赫数：$Ma_{local} = |v_{local}| / c_{local}$。

### 3.2 性能与架构优化

1.  **消除冗余计算**：
    *   **Primitive 计算后置**：将 `computePrimitives` 从 `step` 头部移至尾部（BC 应用之后）。
        *   收益：`computeStableTimeStep` 可直接复用上一步结果，无需重复计算。每步节省 1 次全场计算。
    *   **粘性核函数融合**：将 `computeViscosity` + `computeStressTensor` + `computeHeatFlux` + `diffusionStep` 合并为单个核函数调用 `fusedViscousDiffusionKernel`，使用共享内存加载 u/v/T（32×8 + halo=2），应力张量和热通量在寄存器中就地计算、就地消费。
        *   收益：消除 $\tau$ 和 $q$ 中间数组的全局内存往返读写，减少 3 次内核启动开销。
    *   **Update 核函数简化**：`updateKernel` 不再从守恒量反算原始变量，改为直接传入现成的 $u, v, p$。
        *   收益：每网格节省 10+ 次浮点除法与开方运算。
    *   **Flux 核函数清理**：移除仅传入但未使用的守恒量指针参数（`rho_u, rho_v, E, rho_e, T`）。
2.  **内存管理**：
    *   **持久化归约缓冲区**：`d_reduction_buffer_` 改为按 CUB 查询结果动态分配，复用于所有归约操作（温度、粘性、马赫数、波速）。
        *   收益：消除每步 3-4 次 `cudaMalloc/cudaFree` 调用。
    *   **锁页内存流水线**：`h_pinnedReduction_` 锁页内存用于 `computeStableTimeStep` 异步归约，消除 CPU-GPU 同步停顿。

### 3.3 2026/3 SSP-RK2 时间积分 + GPU 优化

将时间积分从一阶欧拉前进升级为 **SSP-RK2（Strong Stability Preserving Runge-Kutta 2nd order）**，并对核心核函数实施 GPU 级别的访存优化。

#### 3.3.1 SSP-RK2 时间积分

一阶欧拉前进在 MUSCL 二阶空间重构下存在稳定性不匹配（空间二阶 + 时间一阶），导致尾流区产生棋盘噪声。SSP-RK2 将时间精度提升至二阶。

$$
\begin{aligned}
\mathbf{U}^* &= \mathbf{U}^n + \Delta t \cdot \mathbf{L}(\mathbf{U}^n) \\
\mathbf{U}^{**} &= \mathbf{U}^* + \Delta t \cdot \mathbf{L}(\mathbf{U}^*) \\
\mathbf{U}^{n+1} &= \frac{1}{2}\mathbf{U}^n + \frac{1}{2}\mathbf{U}^{**}
\end{aligned}
$$

实现需要三组缓冲区：
- `d_rho_` 等：$\mathbf{U}^n$（当前步，也是最终输出 $\mathbf{U}^{n+1}$）
- `d_rho_new_` 等：$\mathbf{U}^*$（Stage 1 输出）
- `d_rho_rk_` 等：$\mathbf{U}^{**}$（Stage 2 输出）

#### 3.3.2 GPU 访存优化

1. **线程块形状 32×8**：所有 2D 核函数从 `dim3(16, 16)` 改为 `dim3(32, 8)`。X 维度对齐 warp 宽度（32），在 SoA 布局下每个 warp 访问连续 128B cache line。
2. **Shared Memory Tiling（computeFluxesKernel）**：MUSCL ±2 stencil 在 32×8 block 内每个数据点被 ~4.5 次邻居读取。将 rho/u/v/p/cell_type 预加载到共享内存（含 halo=2），消除重复全局显存访问。共享内存用量约 8.6 KB/block。
3. **除法消除（updateKernel）**：预计算 `inv_dx = 1.0f / dx`、`inv_dy = 1.0f / dy`，20 处除法替换为乘法。

### 3.4 当前计算链 (CFDSolver::step)

```cpp
void CFDSolver::step(SimParams &params)
{
    // 注意: 原始变量(d_u_, d_v_, d_p_, d_T_)在进入此函数时已由
    //       reset() 或上一步 step() 结尾的 computePrimitives 计算完毕

    if (params.enable_viscosity)
    {
        // ========== Navier-Stokes求解器(Strang算子分裂 + SSP-RK2对流) ==========

        // 步骤1: 扩散半步(dt/2)
        launchFusedViscousDiffusionKernel(u, v, T, mu, rho_u, rho_v, E, rho_e,
                                          cell_type, dt*0.5, dx, dy, nx, ny);

        // 步骤2: SSP-RK2 对流全步
        launchComputePrimitivesKernel(...)  // 刷新被扩散半步修改的原始变量

        // Stage 1: U* = U^n + dt * L(U^n) → d_rho_new_
        launchComputeFluxesKernel(rho, u, v, p, cell_type → flux_*_x/y)
        launchUpdateKernel(rho, ..., u, v, p, flux_* → rho_new, ...)
        launchApplyBoundaryConditionsKernel(rho_new, ...)
        launchComputePrimitivesKernel(rho_new, ... → u, v, p, T)

        // Stage 2: U** = U* + dt * L(U*) → d_rho_rk_
        launchComputeFluxesKernel(rho_new, u, v, p, cell_type → flux_*_x/y)
        launchUpdateKernel(rho_new, ..., u, v, p, flux_* → rho_rk, ...)

        // Blend: U^{n+1} = 0.5 * U^n + 0.5 * U**
        launchSSPRK2BlendKernel(rho, ..., rho_rk, ...)  // 覆写 d_rho_

        // 步骤3: 扩散半步(dt/2)
        launchComputePrimitivesKernel(...)
        launchFusedViscousDiffusionKernel(u, v, T, mu, rho_u, rho_v, E, rho_e,
                                          cell_type, dt*0.5, dx, dy, nx, ny);
    }
    else
    {
        // ========== 欧拉求解器(无粘性, SSP-RK2) ==========
        // 原始变量已由上一步结尾计算完毕

        // Stage 1: U* = U^n + dt * L(U^n) → d_rho_new_
        launchComputeFluxesKernel(rho, u, v, p, cell_type → flux_*_x/y)
        launchUpdateKernel(rho, ..., u, v, p, flux_* → rho_new, ...)
        launchApplyBoundaryConditionsKernel(rho_new, ...)
        launchComputePrimitivesKernel(rho_new, ... → u, v, p, T)

        // Stage 2: U** = U* + dt * L(U*) → d_rho_rk_
        launchComputeFluxesKernel(rho_new, u, v, p, cell_type → flux_*_x/y)
        launchUpdateKernel(rho_new, ..., u, v, p, flux_* → rho_rk, ...)

        // Blend: U^{n+1} = 0.5 * U^n + 0.5 * U**
        launchSSPRK2BlendKernel(rho, ..., rho_rk, ...)
    }

    // 应用边界条件
    launchApplyBoundaryConditionsKernel(rho, ...)

    // 预计算下一帧所需的原始变量（可视化、CFL、下一步通量计算）
    launchComputePrimitivesKernel(rho, rho_u, rho_v, E, rho_e → u, v, p, T)

    // 更新仿真时间
    params.t_current += params.dt;
    params.step++;
}
```
